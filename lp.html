RAGNAROK ONLINE LEDGER

RagnarokOnlineの売買、清算、資金予定をまとめて管理するブラウザアプリ

pROfitiaは、RagnarokOnlineのゲーム内取引を想定した個人用の収支管理支援アプリです。 収入、支出、清算、売却予定、購入予定、検索、週間・月間の収支確認をブラウザ上で扱えます。 アイテム名、価格、数量、タグを中心に記録し、過去の売買状況や金策の傾向を後から確認しやすくします。

主な機能

  • 収入、支出、清算の3モードで明細を登録
  • アイテム価格と数量から金額を自動計算
  • グループ構成によるアイテム選択の省力化
  • #タグによる明細分類、検索、収支確認
  • 売却予定、購入予定、可用資金によるキャッシュフロー管理
  • 期間、区分、タグ、アイテム名による明細検索
  • 週間、月間の収支推移と明細分布の確認
  • グループ、アイテム、グループ構成、色テーマ、バックアップ管理

FEATURES

各機能の役割

01

明細入力

日々のゲーム内取引を登録する中心機能です。収入、支出、清算の3モードを切り替えて、 アイテム/項目、価格、数量、タグを入力します。入力後のデータは収支取引概要と明細ログに反映されます。

ENTRY FORM
収入支出清算
日付2026/05/14今日
タグ#装備更新 #投資
アイテム/項目メテオライトの欠片
販売価格27,000,000
数量700
  • 収入は販売価格、支出は購入価格として扱います。
  • 価格と数量から金額を自動計算します。
  • 1K、100M、2G、1T のような単位入力に対応しています。
  • アイテム直入力時は、過去に使ったアイテム名が候補表示されます。
  • グループを展開すると、グループに紐づいたアイテムを選択できます。
  • タグは # から始まるチップ形式で複数付与できます。
  • 明細ログをクリックすると、登録済み明細を後から編集できます。
02

清算モード

複数アイテムの売却額を人数で分配する時に使います。清算明細をまとめて入力し、 人数から1人あたりの配布額を確認できます。買取が発生して配布額が一律にならない場合も、 買取者ごとの配布金額を確認してから明細へ追加できます。

清算明細カード帖 90個11.34M
買取明細Aさん 買取 50K-50K
人数3
通常3.78M
Aさん-46.22M
  • 清算明細は複数行を一括登録できます。
  • 清算明細の合計から人数分の配布金額を計算します。
  • 買取明細では買取者、買取価格、数量を入力できます。
  • 配布金額表で追加対象にチェックしたグループが、登録対象になります。
  • 追加時は清算明細を収入、買取明細を支出として登録できます。
  • 各行の計算アシストから価格や数量を逆算できます。
03

予定入力

売却予定と購入予定を登録して、今後のキャッシュフローを確認する機能です。 可用資金を基準に、売る予定のアイテム、買う予定のアイテム、差引予定額を明示的に確認できます。

  • 明細へ送ると、売却予定は収入、購入予定は支出として登録画面に送られます。
04

明細検索

登録済み明細を、期間、収入/支出区分、タグ、アイテム/項目名で検索できます。 過去に何をいくらで売買したか、特定タグの収支がどうだったかを確認するための画面です。

ITEM SEARCH
2026/05/01〜2026/05/31すべて
タグ#イベント #投資
日付区分タグ金額 05/09収入#イベント18.9M 05/08支出#装備-2.1G
  • 開始日と終了日を任意の日付で指定できます。
  • 収入、支出、すべての区分で絞り込みできます。
  • タグは複数指定でき、いずれかのタグを含む明細がヒットします。
  • 検索結果には収入合計、支出合計、収支合計が表示されます。
  • 検索結果の行をクリックすると、その明細を編集できます。
  • 検索結果表にはソート機能があります。
05

収支確認

週間または月間で集計した収支状況を確認する画面です。 収入、支出、収支の推移をグラフで確認し、選択期間のタグ別・アイテム別の明細分布も確認できます。

PERFORMANCE BOARD
収入支出収支目標収支
  • 週間と月間を切り替えて収支を確認できます。
  • 週間は火曜日12:00から翌火曜日11:59までを1期間として扱います。
  • 月間は年と月を選択して表示できます。
  • 週間表示では過去12週間、月間表示では過去12か月の推移を確認できます。
  • 収入/支出軸と収支軸は別軸で表示されます。
  • 目標収支ライン、軸上限、タグ別・アイテム別分布を確認できます。
06

設定

グループ、アイテム、グループ構成、色テーマ、バックアップを管理する画面です。 よく使うアイテムや固定相場で扱えるアイテムを登録しておくと、明細入力時の入力負荷を減らせます。

WORKFLOW

基本的な使い方

1. 必要なマスタを登録する

設定タブで、よく使うグループとアイテムを登録します。アイテムに販売価格を設定しておくと、 明細入力でアイテムを選んだ時に価格が自動反映されます。

2. グループ構成を作る

管理するアイテムが多い場合は、グループ構成でグループとアイテムを紐づけます。 明細入力時にグループを展開すると、対象グループ内のアイテムだけを選択できます。

3. 明細を登録する

収入、支出、清算を選んで明細を登録します。仮登録として先に入れておき、 後から明細ログや検索結果から編集する運用もできます。

4. 予定と収支を確認する

予定入力で売却予定と購入予定を管理し、収支確認で週間・月間の推移を確認します。 金策の過不足、支出の膨張、タグ別の傾向確認に活用できます。

DATA STORAGE

データ保存とバックアップについて

pROfitiaの登録データは、基本的に使用中のブラウザ内に保存されます。 サーバーに自動保存される仕組みではないため、Cookie削除、履歴削除、ブラウザデータ削除、 端末変更などによってデータが消える可能性があります。

定期的に設定タブのエクスポート機能でバックアップJSONを保存してください。 復元したい場合は、同じく設定タブのインポート機能からバックアップファイルを読み込めます。 GitHub Pagesで公開して使う場合も、データは利用者ごとのブラウザに保存されます。
